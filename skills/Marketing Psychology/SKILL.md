## 1. Foundational Thinking Models

### First Principles
Break problems down to fundamental truths, then build up.
- "What must be true for this to work?"

### Jobs to Be Done
People buy solutions to problems, not products.
- "What job is the customer hiring this product to do?"

### Inversion
Solve by avoiding failure instead of seeking success.
- "What would guarantee failure? Now avoid that."

### Pareto Principle (80/20)
80% of results come from 20% of efforts.
- Focus on the vital few, not the trivial many.

### Second-Order Thinking
Consider consequences of consequences.
- "And then what happens?"

### Opportunity Cost
Every choice means giving up alternatives.
- "What am I NOT doing by doing this?"

## 2. Understanding Buyers

### Mere Exposure Effect
Familiarity breeds preference.
- **Application:** Consistent brand presence, retargeting, content frequency

### Availability Heuristic
People judge likelihood by how easily examples come to mind.
- **Application:** Share customer stories frequently

### Confirmation Bias
People seek info that confirms existing beliefs.
- **Application:** Align messaging with what prospects already believe

### Sunk Cost Fallacy
People continue because of past investment.
- **Application:** Remind users of their progress/investment

### Endowment Effect
People value things more once they own them.
- **Application:** Free trials, "try before you buy"

### IKEA Effect
People value things they helped create.
- **Application:** Customization, onboarding that involves input

### Zero-Price Effect
"Free" is irrationally powerful.
- **Application:** Freemium, free shipping thresholds

### Paradox of Choice
Too many options leads to paralysis.
- **Application:** Limit options, highlight recommended choice

### Peak-End Rule
Experiences judged by peak moment and ending.
- **Application:** Nail the onboarding and offboarding moments

### Zeigarnik Effect
Incomplete tasks are remembered better.
- **Application:** Progress bars, "almost there" messaging

## 3. Influencing Behavior (Cialdini's Principles)

### Reciprocity
People feel obligated to return favors.
```
Give value first:
- Free resources, tools, templates
- Helpful content without asking
- Generous free tier
```

### Commitment & Consistency
Small commitments lead to larger ones.
```
Start small:
- Email signup → free trial → paid
- Micro-commitments build momentum
```

### Social Proof
People follow others' actions.
```
Show evidence:
- "Join 10,000+ teams"
- Customer logos
- Testimonials with specifics
- Real-time activity ("Sarah just signed up")
```

### Authority
People defer to experts.
```
Establish credibility:
- Credentials, certifications
- Media mentions
- Expert endorsements
- Data and research
```

### Liking
People say yes to those they like.
```
Be likable:
- Founder story
- Behind-the-scenes
- Personality in brand voice
- Similar to the audience
```

### Scarcity
Limited availability increases desire.
```
Create urgency:
- Limited time offers
- Limited spots
- Exclusive access
- Genuine constraints only
```

### Unity
People favor their "tribe."
```
Create belonging:
- Community membership
- Shared identity ("for founders")
- Us vs. them positioning
```

## 4. Pricing Psychology

### Anchoring
First number seen influences perception.
```
Show high anchor first:
- Enterprise tier on left
- "Usually $199, now $99"
- Compare to expensive alternatives
```

### Decoy Effect
Add option to make target look better.
```
Three-tier pricing:
- Basic: $10
- Pro: $25 ← target
- Enterprise: $100

Add decoy:
- Basic: $10
- Plus: $22 (decoy - bad value)
- Pro: $25 ← now obviously best
```

### Charm Pricing
$99 feels significantly less than $100.
- Use for consumer products
- Avoid for luxury/premium positioning

### Rule of 100
- Under $100: Use percentage ("20% off")
- Over $100: Use absolute ("$50 off")

### Price Relativity
Context determines perceived value.
```
"$50/month is less than your daily coffee"
"$600/year = $1.64/day"
```

## 5. Design & Delivery

### Hick's Law
More choices = longer decision time.
- Reduce options on key pages
- Progressive disclosure

### AIDA Funnel
**A**ttention → **I**nterest → **D**esire → **A**ction
```
Headline (Attention)
↓
Problem/Solution (Interest)
↓
Benefits/Proof (Desire)
↓
CTA (Action)
```

### BJ Fogg Behavior Model
Behavior = Motivation × Ability × Trigger
```
Increase conversion:
- Boost motivation (benefits, urgency)
- Increase ability (simplify, reduce friction)
- Time trigger right (when motivated + able)
```

### EAST Framework
Make desired action:
- **E**asy — Remove friction
- **A**ttractive — Make it appealing
- **S**ocial — Show others doing it
- **T**imely — Right moment

### Nudge Theory
Small design changes guide behavior without restricting.
```
- Default to annual billing
- Pre-select recommended option
- Progress indicators
```

## 6. Growth & Scaling

### Feedback Loops
Positive: More users → more value → more users
Negative: Complaints → churn → fewer users → less development

### Network Effects
Product gets more valuable as more people use it.
- Direct: Communication tools
- Indirect: Platforms (more sellers → more buyers)

### Flywheel Effect
Small wins compound into momentum.
```
Content → Traffic → Leads → Customers →
Case studies → More content...
```

### Switching Costs
Investment that makes leaving painful.
- Data accumulated
- Integrations set up
- Team trained
- Workflows built

### Critical Mass
Minimum users needed for network effects.
- Focus on one niche first
- Geographic concentration

## Quick Application Guide

| Goal | Techniques |
|------|------------|
| Increase signups | Social proof, free trial, reduce friction |
| Raise prices | Anchoring, decoy, value framing |
| Reduce churn | Sunk cost, IKEA effect, switching costs |
| Create urgency | Scarcity, loss aversion, deadlines |
| Build trust | Authority, consistency, reciprocity |
| Improve conversion | Simplify choices, EAST, nudges |
| Grow virally | Network effects, referral program |

## Ethical Guidelines

- Use genuine scarcity only
- Be honest about social proof
- Don't manufacture fake urgency
- Respect user autonomy
- Create real value first
