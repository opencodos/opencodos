# /eod — End of Day Summary

> Synthesize all calls and messages from today into actionable next steps.

## When to Use

- End of a call-heavy day
- Before evening to capture action items
- When you need to see what happened today across all channels

## Input Sources

1. **Granola transcripts** — All calls from today (`1 - Inbox/Granola/*{today}*`)
2. **Telegram daily summary** — `1 - Inbox/Telegram/Daily Summary/{today}.md`

## Execution

### Step 1: Find Today's Data

```bash
# Find today's Granola transcripts
ls -la "$VAULT_PATH/1 - Inbox (Last 7 days)/Granola/" | grep $(date +%Y-%m-%d)

# Confirm Telegram summary exists
cat "$VAULT_PATH/1 - Inbox (Last 7 days)/Telegram/Daily Summary/$(date +%Y-%m-%d).md"
```

### Step 2: Launch Parallel Subagents (Opus)

For **each Granola transcript**, launch a Task subagent with this prompt:

```
Read and summarize this call transcript: [PATH]

Extract:
1. Who is this person / what's their company
2. Company size and context
3. What problem do they want solved
4. Qualification: budget, timeline, decision maker?
5. Outcome - qualified? Next steps?
6. Specific action items for the user

Be concise. Output in markdown.
```

For **Telegram summary**, launch one Task subagent:

```
Read today's Telegram messages: [PATH]

Extract:
1. Key updates from key people (team, cofounders, clients, etc.)
2. New leads that came in
3. Action items requiring response

Be concise. Output in markdown.
```

**Important**: Use `model: opus` for all subagents.

### Step 3: Orchestrate Results

Once all subagents complete, synthesize into final output:

1. **Executive summary** (2-3 sentences)
2. **Action steps by call/person** — Each person gets:
   - Status (qualified/disqualified/needs follow-up)
   - Table of action items with owner and deadline
3. **Summary stats** — Calls completed, leads qualified, etc.

### Step 4: Confirm with User

Present the summary and ask:
- "Are all action items captured correctly?"
- "Any items to add/remove/change?"
- "How can I help right now?" + offer specific tasks

### Step 5: Save Output

Save to: `$VAULT_PATH/0 - Daily Briefs/End of Day/{today}.md`

### Step 6: Update CRM

After saving the EOD summary, update the CRM with today's interactions:

**1. Update `contacts.yaml`** (`$VAULT_PATH/4 - CRM/contacts.yaml`)

For each person from today's calls:
- If exists: update `last_connection`, `next_step`, `deal_stage` (if changed)
- If new contact: add entry with all known fields

**2. Update/Create Profiles** (`$VAULT_PATH/4 - CRM/Profiles/`)

For each person:
- If profile exists: append to Interaction Log section with today's summary
- If no profile: create `{Name}.md` with Quick Facts + today's interaction

**Profile update format:**
```markdown
## Interaction Log

### {date} — {call type}
- **Context**: [Company, role, why they reached out]
- **Outcome**: [Qualified/Disqualified/Follow-up needed]
- **Key points**: [2-3 bullet points]
- **Next steps**: [Action items with owners]
```

**contacts.yaml update fields:**
```yaml
- last_connection: "{today}"
- next_step: "[From action items]"
- deal_stage: "[discovery/qualified/proposal/negotiation/closed]"
```

## Output Format

```markdown
# End of Day Summary — {date}

## Executive Summary
[2-3 sentence summary of the day]

---

## Action Steps by Call/Person

### 1. [Person Name] ([Company — context])
**Status**: [Qualified/Disqualified/Needs follow-up]

| Action | Owner | Deadline |
|--------|-------|----------|
| [Action item] | [Owner] | [Deadline] |

---

[Repeat for each call]

---

## Summary Stats

- **Calls completed**: X
- **High-quality leads**: X (names)
- **Disqualified**: X
- **Needs follow-up**: X

---

*Generated by Atlas /eod — {date}*
```

### Step 7: Share Lead Calls to Team Vault

After EOD is saved and CRM updated, offer to share relevant call notes with the team.

> **Paths**: Read `~/.atlas/paths.json` to resolve `vaultPath` and `sharedVaultPath`. All paths below use these variables. Never hardcode user-specific paths.

**1. Resolve paths**

```
paths = read ~/.atlas/paths.json
VAULT = paths.vaultPath
SHARED = paths.sharedVaultPath
LEADS_FILE = {VAULT}/1 - Inbox (Last 7 days)/Telegram/Leads Bot.md
NOTES_DIR = {SHARED}/Meeting Notes
```

**2. Match today's calls against leads**

- Read `{LEADS_FILE}` and extract all lead emails (lines matching `> Email: ...`)
- List today's Granola meetings via `list_meetings` (time_range: `this_week`, filter to today)
- Match: if any meeting participant email appears in the leads email set → it's a lead call

**3. Present checklist to user**

Use `AskUserQuestion` with multiSelect to show matched calls:

```
Which lead calls should I share to the team vault?

- [x] Jane Smith (Acme Corp) — 1:00 PM
- [x] Bob Lee (WidgetCo) — 4:30 PM
- [ ] Alice Chen — 2:30 PM
```

Pre-select all matched calls. User can deselect any they want private.

**4. Export confirmed calls**

For each confirmed call:
- Fetch the full meeting summary via `get_meetings` (use meeting ID)
- **Sanitize the filename** (see rules below)
- Write to: `{NOTES_DIR}/{sanitized_filename}.md`
- Format:

```markdown
# {Person Name} ({Company}) — {date}

## Participants
- [list]

## Summary
[AI-generated summary from Granola]

## Key Points
[extracted from notes]

## Action Items
[from EOD analysis]

---
*Synced by Atlas /eod*
```

**Filename sanitization rules:**

```
Input:  "{YYYY-MM-DD} - {Person Name} ({Company})"

1. Transliterate Cyrillic → Latin (e.g., Иван Петров → Ivan Petrov)
2. Remove characters not in [a-zA-Z0-9 ._()-]
3. Replace multiple spaces with single space
4. Trim whitespace
5. Max 100 chars (truncate, don't break words)

Examples:
  "2026-02-06 - Jane Smith (Acme Corp)"
  "2026-02-06 - Bob Lee (WidgetCo)"
  "2026-02-06 - Alice Chen (Consulting Inc)"
```

**5. Git commit + push**

```bash
cd {SHARED}
git add "Meeting Notes/"
git commit -m "Add call notes: {comma-separated names} — {date}"
git push
```

**6. Confirm to user**

> Shared {N} call notes to team vault.

## Notes

- Skip calls with incomplete/truncated transcripts (no action items)
- Telegram action items should only be included if clearly actionable and correct
- When in doubt, ask user to confirm before finalizing
- Always offer to help with specific tasks at the end
- Step 7 (team vault sync) only runs if leads file exists and Granola meetings are found
- All paths resolve from `~/.atlas/paths.json` — never hardcode absolute paths in this skill
